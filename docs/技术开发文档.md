## 二、交付物设计

### 2.1 三个应用

每个租户可获得以下三个应用：

| 应用 | 说明 | 部署方式 | 预览方式 |
|------|------|----------|----------|
| **门户** | 租户对外展示的网站 | 动态渲染 | 浏览器直接打开 |
| **管理后台** | 租户的后台管理系统 | 动态菜单 | 浏览器直接打开 |
| **小程序** | 租户的小程序 | 需发布到微信 | H5 模拟 + 微信开发者工具 |

### 2.2 核心特性

- **配置即上线**：在管理后台配置页面、发布即可，无需重新编译部署
- **动态渲染**：页面配置存库，访问时动态渲染
- **数据隔离**：租户级别的数据、权限、资源隔离

### 3.2 tenant-runtime（运行时服务）

#### 3.2.1 核心能力

| 能力 | 说明 |
|------|------|
| 门户动态渲染 | 根据租户配置动态渲染页面 |

#### 3.2.2 门户渲染机制

```javascript
// 动态组件渲染示例
const PageRenderer = {
  props: ['pageConfig'],
  setup(props) {
    return () => h('div', 
      props.pageConfig.components.map(comp => 
        h(resolveComponent(comp.name), comp.props)
      )
    )
  }
}
```

## 四、鉴权设计

### 4.1 鉴权方式

ai-editor 和 tenant-runtime 复用 renren-security 的鉴权方式：

| 机制 | 说明 |
|------|------|
| JWT Token | 登录后颁发 Token，Header 传递 |
| 请求拦截 | 拦截请求校验 Token 有效性 |
| 权限注解 | 使用 `@RequiresPermissions` 控制接口权限 |
| 数据隔离 | 所有查询自动注入 `tenant_id` |

### 4.2 权限体系

参考 renren-admin 的 RBAC 体系：

```
角色 (Role)
    │
    ├── 平台管理员 (platform_admin)
    │   └── 管理所有租户、AI配置、全局设置
    │
    ├── 租户管理员 (tenant_admin)
    │   └── 管理本租户的项目、模板、页面
    │
    └── 编辑人员 (editor)
        └── 使用 AI 生成页面、管理模板
```

## 五、部署方案

### 5.1 混合部署模式

根据客户规模和需求，采用不同部署方式：

| 客户类型 | 方案 | 占比 | 数量 |
|----------|------|------|------|
| 小客户 | SaaS 共享服务 | 80% | 8000家 |
| 中客户 | 独立实例 | 15% | 1500家 |
| 大客户 | 私有化部署 + 远程控制 | 5% | 500家 |

### 5.2 SaaS 共享服务

- 所有小客户共享同一套服务
- 重启影响所有租户
- **解决方案**：
  - 滚动更新（先更新一部分，再更新另一部分）
  - 蓝绿部署（两套环境切换）
  - 灰度发布（只影响小部分用户测试）

### 5.3 私有化部署 + 远程控制

```
┌─────────────────┐   ┌─────────────────┐   ┌─────────────────┐
│   租户A服务器    │   │   租户B服务器    │   │   租户C服务器    │
│  ┌───────────┐  │   │  ┌───────────┐  │   │  ┌───────────┐  │
│  │ 门户服务   │  │   │  │ 门户服务   │  │   │  │ 门户服务   │  │
│  │ 后端服务   │  │   │  │ 后端服务   │  │   │  │ 后端服务   │  │
│  └───────────┘  │   │  └───────────┘  │   │  └───────────┘  │
│      ↓          │   │      ↓          │   │      ↓          │
│   远程控制 ◄────┴───┼─── 远程控制 ◄────┴───┼─── 远程控制     │
│        │           │        │           │        │           │
│        └───────────┼────────┴───────────┼────────┘           │
│                    ▼                                           │
│         ┌──────────────────────┐                              │
│         │   控制中心（你的服务器）│  ← 远程管理、配置下发       │
│         └──────────────────────┘                              │
└─────────────────────────────────────────────────────────────┘
```

### 5.4 远程控制能力

| 能力 | 说明 |
|------|------|
| 配置下发 | 租户服务动态加载配置，无需重启 |
| 版本升级 | 容器/脚本升级，可设置维护时间 |
| 监控告警 | 租户服务状态上报 |
| 数据汇总 | 租户数据定期同步 |

**核心：配置动态加载**
- 租户服务启动时从控制中心拉配置
- 配置变更推送到租户服务
- 租户服务无需重启即可生效

## 六、数据库设计

### 6.1 设计位置

- **renren_security**：用户、部门、岗位、角色、权限、菜单等基础表 + 租户表
- **ai_editor**：AI 配置相关业务表
- **tenant_runtime**：运行时数据（发布的页面、菜单等）

### 6.2 表设计原则

1. **多租户隔离**：所有业务表包含 `tenant_id`
2. **软删除**：使用 `deleted` 标记
3. **公共模块复用**：用户、角色等直接用 renren_security 的表
