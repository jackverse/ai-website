# AI 配置需求文档

## 一、模块概述

AI 配置是整个平台的核心能力，通过 AI 能力自动生成各类应用页面和接口。

## 二、模型支持

### 2.1 多模型接入

- **支持接入多个 AI 模型**，灵活切换
- 兼容 OpenAI GPT 系列
- 兼容 Claude 系列
- 支持国产模型（通义、文心等）
- 支持本地模型（Ollama 等）

### 5.3 联动页面跳转逻辑

> **多个有跳转需求的页面之间的跳转关系设计**

#### 方案设计

| 方案 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| **AI 自动推断** | AI 根据页面内容自动推断跳转关系 | 简单，用户无感知 | 可能不准确 |
| **路由约定** | 通过命名约定自动建立跳转（如 list→detail） | 无需配置 | 灵活性差 |
| **手动配置** | 生成后手动配置跳转关系 | 精准 | 费时 |
| **混合模式** | AI 推断 + 手动调整 | 平衡 | - |

#### 推荐方案：混合模式

1. **生成时 AI 推断**：AI 根据页面语义自动推断跳转关系，写入元数据
2. **预览时验证**：预览时可点击跳转，验证是否正确
3. **手动调整**：支持手动添加/修改跳转关系
4. **路由自动生成**：根据跳转关系自动生成路由配置

#### 跳转关系定义

```typescript
interface PageLink {
  sourcePage: string;    // 源页面ID
  targetPage: string;    // 目标页面ID
  trigger: 'button' | 'card' | 'list-item' | 'form-submit';  // 触发方式
  event: 'click' | 'submit';  // 事件类型
  params?: Record<string, string>;  // 传递参数
}
```

#### 示例

```
生成"商品列表页"和"商品详情页"时：
- AI 自动推断：列表页点击商品卡片 → 详情页
- 自动生成跳转关系：
  - trigger: 'card'
  - event: 'click'  
  - params: { id: 'productId' }
```

## 六、修改方式

### 6.1 自然语言修改

- 通过描述需求修改页面
- 如："把标题改成蓝色"、"增加一个搜索框"
- AI 理解上下文，局部修改

### 6.2 区域坐标修改

- 页面提供**截图/标注功能**
- 用户可以在页面上**框选指定区域**
- 系统获取区域的坐标值（x, y, width, height）
- 将坐标区域 + 修改需求发送给 AI
- AI 根据坐标精确定位修改区域

```
交互流程：
1. 点击"区域修改"按钮
2. 在页面上框选要修改的区域
3. 获得选区坐标值
4. 描述修改需求（如"把这里改成红色"）
5. AI 根据坐标定位并修改
6. 预览修改结果
```

### 6.3 代码级修改

- 直接编辑生成的 Vue 代码
- 适合开发者深度定制

## 七、用户流程

### 7.1 页面生成流程

```
1. 选择模型（可选，默认使用配置的首选模型）
2. 输入需求描述（自然语言）
3. AI 解析需求意图
4. 选择页面类型（首页、列表页、详情页、表单页等）
5. 选择风格主题
6. AI 生成 Vue3 + Tailwind + Headless 代码
7. 预览（像真正页面一样）
8. 调整（如需要）
   - 方式A：描述修改需求
   - 方式B：选中区域描述修改
   - 方式C：直接编辑代码
9. 保存到模板库（选择分类）
10. 发布
```

### 7.2 模板管理流程

```
1. 创建模板 → 填写基本信息（名称、分类、标签）
2. 编辑模板 → 修改代码或配置
3. 预览模板 → 查看效果
4. 发布模板 → 可供其他项目使用
```

## 八、技术要求

- 支持多模型接入和灵活切换
- 生成响应时间 < 10 秒
- 支持上下文记忆
- 生成结果可编辑、可保存
- 模板库支持分类和检索
