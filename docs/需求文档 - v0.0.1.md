## 一、核心愿景

**一个 AI 驱动的多租户建站平台**，每个租户可以：
- 通过 AI 配置生成并发布**独立门户页面**
- 定制专属的**后台管理系统**
- 配套定制**小程序**
- 灵活配置各应用的**接口权限**
- 安全可控的数据访问

---

## 二、功能架构

### 2.1 AI 配置中心（核心）

| 功能 | 描述 |
|------|------|
| AI 生成门户页面 | 输入需求，AI 自动生成可发布的门户页面 |
| AI 生成管理系统 | 根据业务场景，AI 生成配套的后台管理界面 |
| AI 生成小程序 | 生成对应的小程序页面和交互 |
| AI 配置接口 | 根据功能需求，自动生成/配置 API 接口 |

### 2.2 多租户管理

| 实体 | 说明 |
|------|------|
| 租户 Tenant | 独立运营单位，拥有独立空间 |
| 门户 Portal | 租户的前台展示站点 |
| 管理后台 Admin | 租户的管理系统 |
| 小程序 MiniApp | 租户的小程序应用 |
| 接口 API | 各应用对应的后端接口 |

### 2.3 权限安全

- **租户隔离**：数据、权限完全隔离
- **最小权限原则**：权限不能给的太大，按需分配
- **接口级控制**：每个接口都有独立的权限配置
- **操作审计**：关键操作留痕

## 四、安全防护设计

### 4.1 认证与鉴权

- JWT Token 认证
- RBAC 角色权限控制
- 接口级权限校验
- 操作日志审计

### 4.2 数据安全

- 租户数据完全隔离
- 敏感数据脱敏
- SQL 注入防护
- XSS 防护

### 4.3 权限管控原则

> **核心原则：权限不能给的太大**

- 默认拒绝：未明确授权的请求一律拒绝
- 最小权限：只授予完成任务所需的最小权限
- 审批流程：敏感操作需要上级审批
- 定期审计：定期检查权限分配是否合理
