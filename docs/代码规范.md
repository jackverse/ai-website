## 二、命名规范

### 2.1 Java 命名
- 类名：`UpperCamelCase`，如 `SiteService`、`PageController`。
- 方法名：`lowerCamelCase`，如 `listPages`、`generatePageCode`。
- 变量名：`lowerCamelCase`，常量使用 `UPPER_SNAKE_CASE`。
- 包名：全小写，按业务域分层（`controller/service/dao/entity`）。
- 布尔变量优先 `is/has/can` 前缀，如 `isPublished`。

### 2.2 TypeScript/Vue 命名
- 组件文件名：`PascalCase`（推荐）或 `kebab-case`。
- 组合式函数：`useXxx` 命名，如 `usePageBuilder`。
- 类型定义：接口以业务语义命名，如 `SiteDto`、`PageQuery`。
- 事件命名：`kebab-case`，如 `save-success`。

### 2.3 数据库命名
- 表名：`snake_case`，统一前缀（如 `saas_`）。
- 字段名：`snake_case`，避免保留关键字。
- 索引命名：`idx_` 前缀；唯一索引使用 `uk_` 前缀。

## 三、代码格式规范

### 3.1 Java
- 缩进 4 空格，单行不超过 120 字符。
- 左大括号前保留空格，运算符前后保留空格。
- 避免魔法值，常量统一抽取。
- 公共方法需有 Javadoc，说明参数与返回值语义。

### 3.2 TypeScript/Vue
- 缩进 2 空格，统一使用单引号。
- 语句结尾保留分号。
- 优先使用 `<script setup lang="ts">`。
- 组件 `props`、`emits` 需显式类型定义。
- 不允许 `any` 泛滥，必要时加注释说明原因与替代计划。

### 3.3 SQL
- 关键字大写（`SELECT`、`WHERE`、`ORDER BY`）。
- 条件字段加索引前先评估查询频率与选择性。
- 禁止直接拼接 SQL 字符串，必须参数化。

## 五、接口规范

### 5.1 通用约定
- 路径建议统一前缀：`/api/v1`。
- 使用 REST 风格资源命名（如 `/site/{id}`）。
- 统一响应结构：

```typescript
{
  code: 200,
  msg: 'success',
  data: {}
}
```

### 5.2 分页结构

```typescript
{
  code: 200,
  msg: 'success',
  data: {
    list: [],
    total: 0,
    page: 1,
    pageSize: 10
  }
}
```

### 5.3 错误码建议
- `200`：成功
- `400`：参数错误
- `401`：未认证
- `403`：无权限
- `500`：系统异常

## 六、安全规范

- 禁止硬编码密钥、Token、账号密码。
- 接口必须校验当前用户是否具备资源访问权限（含租户隔离）。
- 所有数据库操作使用参数化查询。
- 所有外部输入做长度、类型、格式校验。
- 前端展示富文本时必须做 XSS 防护。

## 七、日志规范

### 7.1 日志级别
| 级别 | 场景 |
|------|------|
| `DEBUG` | 调试信息 |
| `INFO` | 正常业务节点 |
| `WARN` | 可恢复异常 |
| `ERROR` | 需告警的异常 |

### 7.2 日志内容要求
- 记录关键信息：资源 ID、租户 ID、操作人、耗时。
- 禁止记录密码、密钥、完整 Token 等敏感数据。
- 错误日志必须包含异常堆栈。

```java
log.info("发布页面成功, tenantId={}, pageId={}, operatorId={}", tenantId, pageId, userId);
log.error("AI 生成失败, tenantId={}, requestId={}", tenantId, requestId, e);
```

## 八、异常处理规范

- 业务异常使用统一 `BusinessException`。
- 控制器层不直接 `try-catch` 吞异常，交由全局异常处理器处理。
- 异常消息对外返回可读文案，对内保留详细堆栈日志。

```java
@RestControllerAdvice
public class GlobalExceptionHandler {
    @ExceptionHandler(BusinessException.class)
    public Result<?> handleBusiness(BusinessException e) {
        return Result.error(e.getCode(), e.getMessage());
    }
}
```

## 九、AI 模块专项规范

- Prompt 模板版本化管理，禁止在业务代码中散落硬编码提示词。
- AI 输出必须经过结构校验（JSON Schema 或等价校验）再入库。
- 生成代码落库前执行基础安全检查（如危险脚本片段检测）。
- AI 调用链路记录耗时与失败原因，支持问题回溯。
